# 驾驶陪练系统设计文档

## 一、系统架构设计

### 1.1 整体架构
```
前端层（Vue 3）
├── 学员端（手机端）
├── 教练端（手机端）
└── 管理端（PC端）

后端层（FastAPI）
├── API接口层
├── 业务逻辑层
├── 数据访问层（DAO）
└── 工具层

数据层（SQLLite）
├── 核心业务数据表
└── 系统配置表
```

### 1.2 技术选型
- **前端框架**：Vue 3 + Vite
- **后端框架**：FastAPI
- **数据库**：SQLLite
- **状态管理**：Session（后端） + Pinia（前端）
- **API文档**：FastAPI自动生成OpenAPI文档

## 二、数据库设计

### 2.1 数据库表结构

#### 2.1.1 学员表（students）
```sql
CREATE TABLE students (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL,
    phone VARCHAR(20) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE,
    password VARCHAR(255) NOT NULL,
    area_id INTEGER NOT NULL,
    status VARCHAR(20) DEFAULT '正常',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 2.1.2 区域表（areas）
```sql
CREATE TABLE areas (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 2.1.3 教练表（coaches）
```sql
CREATE TABLE coaches (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL,
    phone VARCHAR(20) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE,
    password VARCHAR(255) NOT NULL,
    area_id INTEGER NOT NULL,
    status VARCHAR(20) DEFAULT '待确认',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 2.1.4 排课表（schedules）
```sql
CREATE TABLE schedules (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    coach_id INTEGER NOT NULL,
    schedule_date DATE NOT NULL,
    schedule_time VARCHAR(20) NOT NULL,
    student_id INTEGER,
    status VARCHAR(20) DEFAULT '待确认',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 2.1.5 管理员表（admins）
```sql
CREATE TABLE admins (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    email VARCHAR(100) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 2.2 DAO设计
每个实体对应一个DAO类，提供CRUD操作：
- StudentDAO
- AreaDAO

- ScheduleDAO
- AdminDAO

## 三、API接口设计

### 3.1 学员接口
- `POST /api/students/register`：学员注册
- `POST /api/students/login`：学员登录
- `GET /api/students/schedules`：获取学员可选课程
- `POST /api/students/schedules/{id}/book`：预约课程

### 3.2 教练接口
- `POST /api/coaches/register`：教练注册
- `POST /api/coaches/login`：教练登录
- `GET /api/coaches/schedules`：获取教练排课信息
- `POST /api/coaches/schedules`：添加排课
- `PUT /api/coaches/schedules/{id}/confirm`：确认课程
- `PUT /api/coaches/schedules/{id}/complete`：标记课程完成

### 3.3 管理员接口
- `POST /api/admins/login`：管理员登录
- `GET /api/admins/areas`：获取区域列表
- `POST /api/admins/areas`：添加区域
- `GET /api/admins/coaches`：获取教练列表
- `PUT /api/admins/coaches/{id}/status`：更新教练状态
- `PUT /api/admins/coaches/{id}/password`：重置教练密码

## 五、状态管理设计

### 5.1 后端Session管理
- 使用FastAPI的SessionMiddleware
- 存储用户登录状态和角色信息
- Session过期时间：24小时

### 5.2 前端状态管理
- 使用Pinia管理前端状态
- 存储用户信息、菜单权限等
- 实现路由守卫，根据角色控制页面访问权限
